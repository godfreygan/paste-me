## 简介

**PasteMe** 是一款原生 macOS 菜单栏应用，在后台静默记录你的剪贴板历史，帮助你：

- 找回不小心覆盖掉的复制内容
- 在多个应用之间快速切换常用文本、图片、文件路径
- 通过全局快捷键呼出选择面板，选中即粘贴

所有数据保存在本地，不上传云端，不收集任何个人信息。

PasteMe 为**纯菜单栏应用**，启动后不会在 Dock 中显示图标。

## 功能特性

### 核心能力

| 功能 | 说明 |
|------|------|
| **自动记录** | 监听系统剪贴板，自动保存文本、图片、文件路径 |
| **菜单栏入口** | 常驻顶部状态栏，点击即可浏览完整历史 |
| **全局快捷键** | 默认 `⌘⇧V` 呼出选择面板，支持自定义组合键 |
| **选中即粘贴** | 在选择面板或菜单栏历史中点击条目，自动写入剪贴板并粘贴到当前光标位置 |
| **置顶收藏** | 常用内容可置顶，不受历史条数限制 |
| **搜索过滤** | 菜单栏面板内支持关键词搜索（选择面板为纯列表） |
| **条数限制** | 默认保留 20 条，可在 10–1000 之间调整，超出自动删除最旧记录 |
| **开机自启** | 可选登录时自动启动 |
| **本地存储** | JSON 文件 + 本地图片缓存，无网络依赖 |

### 支持的内容类型

| 类型 | 示例 |
|------|------|
| 文本 | 普通字符串、代码片段、URL |
| 图片 | 截图、从浏览器或 Finder 复制的图片 |
| 文件 | Finder 中复制的文件/文件夹路径 |

### 界面与交互

- 选择面板采用圆角毛玻璃风格
- 点击面板外部区域自动关闭
- 选择面板支持键盘导航（`↑` `↓` `Enter` `Esc`）
- 菜单栏面板支持悬停显示操作按钮（置顶 / 删除）

## 安装

### 方式一：下载预编译包（推荐）

1. 打开仓库 [Actions 页面](https://github.com/godfreygan/paste-me/actions)
2. 进入最新一次成功的 **Build PasteMe** 构建
3. 在 **Artifacts** 区域下载 `PasteMe-macOS-installer`
4. 解压 ZIP，将 `PasteMe.app` 拖入「应用程序」文件夹

### 方式二：从源码自行编译

详见下方 [从源码构建](#从源码构建) 章节，或阅读 [INSTALL.md](./INSTALL.md)（含 GitHub Actions 云端打包说明）。

### 首次打开（Gatekeeper 提示）

未签名的应用首次启动时，macOS 可能提示：

> Apple 无法验证「PasteMe.app」是否包含可能危害 Mac 安全或泄漏隐私的恶意软件。

这是正常现象。PasteMe 通过 GitHub Actions 构建，**未做 Apple 官方签名和公证**，macOS 会拦截直接双击打开，**不代表应用有毒**。

**解决方法（任选其一）：**

**方法一：Control + 点击打开（推荐）**

1. 在「应用程序」中找到 `PasteMe.app`
2. **Control + 点击**（或右键）→ 选择 **打开**
3. 在弹窗中再次点击 **打开**
4. 之后可直接双击启动，无需重复操作

**方法二：系统设置中允许**

1. 先双击 `PasteMe.app`（会被拦截）
2. 打开 **系统设置** → **隐私与安全性**
3. 向下滚动，找到类似「已阻止 PasteMe.app 使用，因为无法验证开发者」的提示
4. 点击 **仍要打开**

更多说明见 [INSTALL.md](./INSTALL.md)。

## 快速开始

```
1. 启动 PasteMe          → 菜单栏出现剪贴板图标
2. 正常复制任意内容       → 自动加入历史（默认最多 20 条）
3. 按 ⌘⇧V               → 弹出选择面板
4. 点击或 Enter 选择条目  → 自动粘贴到当前应用光标处
5. 点击菜单栏图标         → 浏览完整历史、搜索、置顶
6. 右键菜单栏图标         → 打开设置或退出应用
```

**首次使用「选中即粘贴」前，请授权辅助功能**（见 [权限说明](#权限说明)），否则条目只会写入剪贴板，不会自动粘贴。

## 使用指南

### 菜单栏面板

**左键点击**菜单栏中的 **剪贴板图标** 打开历史列表（纯列表，无底部工具栏）。

```
┌─────────────────────────────────┐
│ 🔍 搜索框                       │
├─────────────────────────────────┤
│ ⭐ 置顶                         │
│  ├ 常用文本…                    │
│  └ ~/Documents/notes.txt        │
├─────────────────────────────────┤
│ 🕐 最近复制                     │
│  ├ 11:20  刚刚复制的内容…       │
│  ├ 11:15  [图片] 128 KB         │
│  └ 11:10  report.pdf            │
└─────────────────────────────────┘
```

- **点击条目**：关闭面板 → 写入剪贴板 → 模拟 `⌘V` 粘贴到当前光标（与选择面板行为一致）
- **悬停条目**：显示 📌 置顶、🗑️ 删除按钮

**右键点击**菜单栏图标打开上下文菜单：

| 菜单项 | 功能 |
|--------|------|
| 设置 | 打开设置窗口 |
| 退出 | 退出 PasteMe |

### 快捷键选择面板

按下全局快捷键（默认 `⌘⇧V`）呼出，面板出现在鼠标附近。再次按下同一快捷键可关闭面板。

- **列表内容**：按时间倒序展示全部历史（置顶项会显示 📌 标记，与菜单栏分区展示不同）
- **单击条目**：关闭面板 → 写入剪贴板 → 模拟 `⌘V` 粘贴到当前光标
- **点击面板外任意位置**：关闭面板
- **键盘操作**：`↑` `↓` 移动高亮，`Enter` 确认，`Esc` 关闭

> 选择面板不包含搜索框，适合快速选取最近复制的内容。需要搜索请使用菜单栏面板。

### 置顶与删除

| 操作 | 方法 | 效果 |
|------|------|------|
| 置顶 | 悬停 → 点击 📌 | 移至置顶区，不会被自动清理 |
| 取消置顶 | 悬停 → 点击 📌（斜线图标） | 回到普通历史 |
| 删除单条 | 悬停 → 点击 🗑️ | 永久删除该条目 |
| 清空历史 | 设置 → 数据 → 清空所有历史记录 | 删除所有非置顶记录 |

### 设置项说明

**右键点击**菜单栏图标 → **设置** 进入设置窗口：

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 最大历史条数 | 非置顶记录的上限 | 20（范围 10–1000） |
| 弹出选择面板 | 点击后按下新快捷键即可录制 | `⌘⇧V` |
| 开机自启动 | 登录 macOS 时自动运行 | 开启 |
| 复制时播放声音 | 菜单栏面板点击条目时的提示音 | 关闭 |

**修改快捷键步骤：**

1. 打开设置 → 点击「弹出选择面板」右侧的快捷键按钮
2. 按下你想要的组合键（须包含 ⌘ / ⌃ / ⌥ / ⇧ 中的至少一个）
3. 按 `Esc` 可取消录制

## 权限说明

PasteMe 需要以下系统权限才能完整工作：

### 辅助功能（自动粘贴必须）

用于模拟 `⌘V`，将选择面板中选中的内容粘贴到前台应用光标处。

**授权路径：**

```
系统设置 → 隐私与安全性 → 辅助功能 → 点击 + → 选择 PasteMe
```

若未授权：选择面板和菜单栏面板仍可复制到剪贴板，但不会自动粘贴（可手动 `⌘V`）。

### 全局快捷键

快捷键通过系统 Carbon API 注册。多数环境下无需额外权限即可使用；若快捷键无响应，请同样检查辅助功能权限，并确认没有其他应用占用相同组合键。

## 数据与隐私

### 存储位置

```
~/Library/Application Support/PasteMe/
├── settings.json    # 用户设置（条数、快捷键、自启等）
├── clips.json       # 剪贴板历史记录
├── pinned.json      # 置顶内容
└── images/          # 图片类历史条目的 PNG 缓存
```

### 隐私承诺

- 所有数据仅保存在本机
- 不连接网络、不上传、不同步 iCloud
- 卸载时删除上述目录即可清除全部数据

### 自动清理规则

1. 新复制内容插入列表最顶部（按时间倒序）
2. 重复内容会去重，保留最新一条
3. 非置顶条目超过「最大历史条数」时，自动删除最旧的记录
4. 置顶条目不占用历史条数配额

## 从源码构建

### 前置条件

- macOS 15.0+
- [Xcode](https://developer.apple.com/xcode/) 16+（完整安装，非仅 Command Line Tools）

### 命令行打包

```bash
git clone git@github.com:godfreygan/paste-me.git
cd paste-me/PasteMe
chmod +x package-xcode.sh
./package-xcode.sh
```

产物位于 `PasteMe/dist/`：

- `PasteMe.app` — 可直接运行
- `PasteMe-1.0.0-macOS.zip` — 可分发安装包

### 使用 Xcode 开发

```bash
cd PasteMe
open PasteMe.xcodeproj
# 在 Xcode 中按 ⌘R 运行
```

### 无本地 Xcode？

可使用 GitHub Actions 云端自动构建，详见 [INSTALL.md](./INSTALL.md)。

## 常见问题

<details>
<summary><strong>复制的内容没有被记录？</strong></summary>

1. 确认菜单栏有 PasteMe 图标（应用正在运行）
2. 确认复制的是文本、图片或文件路径（特殊格式暂不支持）
3. 若刚通过 PasteMe 自身写入剪贴板，会自动忽略以避免重复记录

</details>

<details>
<summary><strong>按 ⌘⇧V 没有反应？</strong></summary>

1. 确认 PasteMe 正在运行（菜单栏有剪贴板图标）
2. 打开设置确认快捷键是否被修改为其他组合
3. 确认没有其他应用占用相同快捷键
4. 若面板能打开但无法自动粘贴，再检查 **辅助功能** 权限（见 [权限说明](#权限说明)）

</details>

<details>
<summary><strong>菜单栏或选择面板中点击条目没有自动粘贴？</strong></summary>

需要 **辅助功能** 权限。PasteMe 会先恢复焦点到之前的应用，再模拟 `⌘V` 实现粘贴。若权限已授予仍无效，请确认目标应用支持标准粘贴操作。

</details>

<details>
<summary><strong>如何彻底卸载？</strong></summary>

1. 退出 PasteMe：打开「活动监视器」，搜索 `PasteMe` 并退出进程；或在终端执行 `killall PasteMe`
2. 将 `/Applications/PasteMe.app` 拖入废纸篓
3. 删除数据目录：

```bash
rm -rf ~/Library/Application\ Support/PasteMe
```

</details>

<details>
<summary><strong>支持 iCloud 同步吗？</strong></summary>

目前不支持。所有数据仅保存在本地。

</details>

<details>
<summary><strong>为什么需要 macOS 15？</strong></summary>

PasteMe 基于 macOS 15 SDK 构建，使用了 Sequoia 时代的系统 API 与 SwiftUI 特性，未对更早版本做兼容测试。

</details>
